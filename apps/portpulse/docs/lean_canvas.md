# PortPulse — Lean Canvas
**/ideate 完了: 2026-04-24**

---

## UVP（確定）

> **「特定バース（石油・LNG 受入基地）のタンカー入港頻度を衛星から自動カウントし、公式輸入統計より 2〜4 週間早くサプライチェーン異常を検知する」**

**スコープ制約（Critic 指摘を踏まえた設計）:**
- 対象: **全長 100m 以上のタンカー・バルカー**（10m/pixel で 10px 以上 = 安定検出可能）
- 個別コンテナ計数は Sentinel-2 10m では不可能（6m × 2.4m はサブピクセル）のため対象外
- 定点 AOI（関心領域）を事前登録し、そのバースの入港頻度変化を時系列でモニタリング

## 1. Problem（課題）

| 指標 | 数値 |
|---|---|
| スエズ運河閉塞 2021年 | $9.6B/日 の物流停止 |
| 日本の石油依存度 | 99% をタンカー輸送（国土交通省） |
| 日本の LNG 輸入量 | 世界最大規模 ~75 百万トン/年。JERA 等が受入 |
| 代替データ市場 | $2.4B/年（ヘッジファンド・商社が支払い意欲あり） |
| 公式統計の遅延 | IEA 月次レポート・METI 貿易統計は 2〜4 週間後 |

- **エネルギートレーダーの課題**: 「在庫が足りなくなる前に」供給変化を察知したい。AIS データは制裁タンカー等の AIS オフ船を追跡できず、統計は後追いになる
- **製造業サプライチェーン**: トヨタ等の JIT 生産は輸入部品依存。特定港の入荷頻度から生産活動変化を公式発表前に把握できれば価値がある（愛知県常滑・舞木のコンテナヤード等）
- **情報の非対称性**: 地上処理・AIS 依存の既存サービスは「見えている船舶のみ」を追跡。衛星直接観測は AIS 非依存

## 4. Unfair Advantage（圧倒的優位性）

| 競合 | 手法 | 限界 | PortPulse 優位性 |
|---|---|---|---|
| Orbital Insight / SpaceKnow | 衛星画像 + 地上 AI | 地上処理のみ。汎用指標で特定バース定点監視なし | **定点 AOI + オンボード処理 + LFM 文脈解釈** |
| Kayrros / OilX | SAR 衛星でタンク在庫レベル監視 | SAR 衛星コスト高。タンク在庫変化のみ（入港動態把握なし） | **Sentinel-2 無償 + タンカー入港動態（在庫変化より先行シグナル）** |
| Bloomberg AIS 集計 | AIS 受信データ統計 | AIS オフ船追跡不可。統計は後追い | **衛星画像直接確認（AIS 非依存）** |
| Synspective / Planet Labs | SAR / 光学衛星の地上処理 | SAR コスト高、光学は地上処理前提 | **Sentinel-2 無償 + LFM でオンボード処理 + 異常の自然言語説明** |

**PortPulse のみが持つ組み合わせ:**
- **LFM 文脈解釈**: 「この入港頻度減少は ① 季節的メンテナンス ② 供給危機 ③ 気象遅延 のどれか」を自然言語で説明。単純カウントとの差別化であり、トレーダーが次のアクションを取れる情報に変換
- **オンボード処理**: 地上局なしで軌道内に異常を検知。地上局接触時に即アラートを送信
- **追加撮像リクエスト（オンボード必然性）**: 異常検知時に同コンステレーション内の隣接衛星へ追加撮像リクエストを中継 → リビジットを 5〜10 日から数時間に短縮。地上処理では不可能な対応速度

## 5. Key Metrics（成功指標）

**精度目標（顧客=エネルギートレーダー・電力会社を想定）:**

| 指標 | 目標値 | 根拠 |
|---|---|---|
| Vessel Detection Recall | ≥ **0.85** | 入港見落としは供給異常の見逃しに直結 |
| False Positive Rate | ≤ **0.15** | 誤アラートが多いとトレーダーの信頼を損なう |
| Precision | ≥ 0.80 | — |
| 先行性（Lead time） | ≥ **14日**（公式統計比） | 代替データとしての価値の核心 |
| 帯域幅削減率 | ≥ **80%** | 変化なしシーンは送信しない |

## 8. Assumptions（前提仮説・未検証）

1. **技術前提**: Sentinel-2 B4/B3/B2 10m で全長 100m 以上のタンカーが信頼性よく検出できる（→ /poc で確認）
2. **滞在時間前提**: 対象バースのタンカー滞在時間が 24〜48 時間以上（5〜10 日リビジットで捕捉可能）
3. **GT 前提**: AIS 歴史データ（Marine Cadastre / MarineTraffic API）から対象バースの入港記録が取得できる
4. **市場前提**: エネルギートレーダーが「公式統計より 2〜4 週間早い」衛星シグナルに月額課金する
5. **AOI 前提**: 対象バースの座標・停泊エリア形状が事前定義されていること（事前設定必須）
6. **LFM 前提**: LFM 2.5-VL-450M が「入港あり/なし」と「異常パターンの文脈判断」を 30 秒以内に実行できる

## エッジ処理の必然性

ハッカソンの与件（"What if satellites had onboard compute?"）のもとで:
- **通常時**: 入港なし = 変化なしシーンを廃棄。地上に 2KB のアラートのみ送信（帯域幅 80% 削減）
- **異常検知時**: 地上局との接触を待たずに同コンステレーション内の別衛星へ「この座標をすぐ撮れ」と中継。リビジット 5〜10 日 → 数時間に短縮
- この「衛星間中継による追加撮像」は地上処理では原理的に不可能（地上局経由では 10〜15 分の接触窓を待つ必要があり、次のパスが来る前に次の行動判断ができない）

## デモシナリオ（固定）

**ストーリー**: 「2022年3月、欧州がロシア産 LNG 調達を急減させた際、フジャイラ港・シンガポール・**京葉シーバース**における LNG タンカー入港頻度が IEA 公式統計より **21 日前**に変化を示した」

| ステップ | Before（PortPulse なし） | After（PortPulse あり） |
|---|---|---|
| 衛星通過 | 全シーン地上へ送信 | LFM が入港有無を判定。変化あり = priority alert |
| 地上局接触 | 生データが数日後に到着。分析に数日追加 | 2KB アラート「入港頻度 -40%。異常パターン：供給不足の可能性」を即送信 |
| トレーダーの対応 | IEA 月次レポートを 2〜4 週間待って対応 | 21 日前にポジション調整・代替調達の意思決定が可能 |

**使用実データ**: 2022年3月の上記港湾座標 → SimSat で Sentinel-2 シーン取得 → LFM で入港頻度変化を検知

## 技術スタック

```
SimSat API (Sentinel-2: B4/B3/B2、定点バース座標)
  → AOI マスク（事前登録バースの停泊エリア）
  → 入港判定: 停泊エリア内に 10px 以上の物体（タンカー）が存在するか
  → 時系列変化率算出: T0〜Tn の入港頻度トレンド
  → LFM 2.5-VL-450M (LoRA FT)
     ※ LFM の役割①: 「入港あり/なし」の視覚確認（誤検知フィルタ）
     ※ LFM の役割②: 頻度変化の文脈解釈「季節的 / 供給危機 / 気象遅延」の分類 + 自然言語説明
  → {"status": "ANOMALY", "frequency_change": -0.42, "interpretation": "供給危機の可能性", "confidence": 0.87}
  → 変化ありの 2KB JSON のみダウンリンク → REST API → トレーディングシステム
```

GT: AIS 船舶到着/出港レコード（Marine Cadastre 歴史 AIS）× SimSat 入港検知の照合
事後検証: IEA/METI 月次輸入統計との lead time 相関（14 日以上の先行性を確認）
