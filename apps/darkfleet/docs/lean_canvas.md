# DarkFleet — Lean Canvas
**/ideate 完了: 2026-04-19**

## /proposer 出力

---

## UVP（確定）

> **「AIS送信を停止した船舶を、地上局なしで衛星通過時に自律検知し、位置・船種・証拠画像を2KBで通報する」**

---

## 1. Problem（課題）

- **違法漁業**: FAO推計で年間 $23B の損失。AIS を意図的にオフにした漁船が公海・EEZ境界で操業
- **制裁逃れタンカー（ダーク艦隊）**: ロシア侵攻後、400隻超のタンカーがAISを無効化し石油輸送を継続。各国制裁当局が追跡不能
- **麻薬密輸・海賊**: AIS 無効化は海洋犯罪の標準手法。国際商業会議所推計で年間 $1〜2B の損害
- **現状の限界**: MarineTraffic 等のAIS集計サービスは「送信していない船舶」を追えない。SAR衛星（e.g. ICEYE）はコストが高く枚数が限られる

---

## 2. Customer Segment / Early Adopter

**Full Customer Segment:**
- 海事保険会社（Lloyd's of London エコシステム）
- トレードファイナンス銀行（貨物担保を確認するため）
- 沿岸警備隊・海軍（違法漁業・制裁監視）
- コモディティトレーダー（石油・穀物の船舶位置確認）

**Early Adopter（現実的な初期顧客）:**
1. **制裁コンプライアンス SaaS 会社**（Windward / Pole Star 等の競合として）: 法的義務が明確。予算あり。API 経由で即活用可能
2. **トレードファイナンス銀行**: 貨物担保の証拠として衛星写真の価値が明確。調達サイクルが比較的短い
3. *沿岸警備・政府は予算はあるが調達が 2〜5 年のため後期フェーズ*

## 4. Unfair Advantage（圧倒的優位性）

| 競合 | 弱点 | DarkFleet の優位性 |
|---|---|---|
| MarineTraffic / VesselFinder | AIS受信のみ。無効化船舶は追跡不能 | **衛星画像で直接検出（AIS依存なし）** |
| ICEYE / Capella（SAR） | SAR衛星数が少なく撮像コスト高い | **Sentinel-2 光学で低コスト、高頻度** |
| Planet Labs | 地上処理前提。証拠画像の生成に遅延 | **オンボード完結 → 法的証拠となる署名付きアラートを軌道内生成** |
| Windward | 行動パターン分析のみ（AIS履歴ベース） | **実画像証拠 + 位置特定で法的証拠力を強化** |

**DarkFleet のみが持つ組み合わせ:**
- **オンボード処理**: 地上局なしで軌道内に証拠画像と座標を確定。改ざん困難
- **VLM 文脈推論**: 「タンカー vs 漁船 vs 岩礁 vs 白波」の識別が閾値処理では不可能
- **AIS照合**: 画像内の船舶とAISデータを突合し「放送なし = ダーク」を判定

**スコープ制約（Critic 指摘を踏まえた設計）:**
- 対象: **港湾・停泊地・狭水道に静止する船舶**（移動中の追跡はリビジットが 5〜10日のため断念）
- 対象船種: 全長 **50m 以上**（10m/pixel で 5px 以上 = 確実に検出可能）
- 港湾・錨地にいる船舶は数日〜数週間滞在するため、5〜10日リビジットで十分

---

## 5. Key Metrics（成功指標）

**精度目標（顧客=コンプライアンス審査官・保険査定員を想定）:**
- Vessel Detection Recall: > **0.80**（静止・大型タンカー対象）
- False Positive Rate: < **0.10**（誤報が多いと法的リスク）
- Precision: > 0.85
- 最小検出対象: 全長 50m 以上の船舶

**技術指標:**
- 推論時間: < 30秒/シーン（衛星通過中に完了）
- アラートサイズ: < 2KB（座標 + 船種判定 + 信頼スコア）
- 空間解像度: 10m（Sentinel-2 B4/B3/B2 RGB）

**AIS照合:**
- AIS オフ判定基準: 衛星検出座標から半径 5km 以内に 24時間以内の AIS Broadcast なし

## 6. Revenue Streams（収益仮説）

- **SaaS サブスクリプション**: 監視海域 or 監視船舶数に応じた月額課金
- **Per-alert 課金**: ダーク船舶検知アラート1件あたり従量課金（スポット顧客向け）
- **コンプライアンスレポート API**: 金融機関向け貨物担保確認 API（KYC/AML 統合）

---

## 7. Channels（チャネル）

1. **主チャネル**: 制裁コンプライアンス SaaS ベンダーへの OEM/API 提供
2. **副チャネル**: Lloyd's 保険エコシステム（ブローカー経由）
3. **長期**: 沿岸警備隊・税関への政府契約

---

## 8. Assumptions（前提仮説・未検証）

1. **技術前提**: Sentinel-2 10m解像度で 50m 以上の船舶を LFM が安定して識別できる（PoC 要検証）
2. **データ前提**: 港湾・錨地の静止船舶に対し SimSat が有効な画像を返す
3. **AIS前提**: AIS受信データ（外部ソース）とのリアルタイム照合が可能。DarkFleet は衛星画像側を提供し、AIS は外部 API と連携
4. **市場前提**: コンプライアンス担当者が「衛星証拠画像付きレポート」に数百万円〜の対価を払う
5. **法的前提**: オンボード生成の証拠画像が法的証拠として採用される（タイムスタンプ・署名の信頼性）
6. **スコープ前提**: 港湾・停泊地の滞在期間が 5 日以上であることが多い（移動中船舶は対象外）

## /critic 評価

### 初回評価（修正前）— 62点推定・要修正
- 致命的弱点①: 漁船（50m未満）が解像度制約で検出困難
- 致命的弱点②: 移動中船舶のリビジット問題（5〜10日）
- → スコープを港湾静止大型タンカーに絞ることを提案

### 修正後評価 — **67点・Go**（スコープ修正後）
スコープ絞り込みで技術的成立性は大幅改善。残課題は PoC レベルに降格。

**強み**
1. **問題の定量性**: 制裁タンカー 400+ 隻・違法漁業 $23B
2. **Early Adopter の調達明確性**: コンプライアンス SaaS は法的義務ドリブンで予算確定
3. **スコープ修正後の整合性**: 50m+ 静止タンカー in 港湾 → 5px 以上・複数日滞在 → リビジット問題解消

**残存課題（PoC フェーズで検証）**
1. **船種分類の解像度限界**: 50m = 5px → タンカー vs 貨物船の識別が困難。UVP を「船舶存在証明 + 座標」にダウングレードする方が確実
2. **FT 用 GT データ**: AIS オフ確認済みタンカーの衛星画像ペアが必要。Reuters/CSIS の制裁タンカー追跡レポートから逆引きして SimSat で取得
3. **AIS 外部依存**: Open AIS API（MarineTraffic free tier / Marine Cadastre）を使ったデモロジックの実装
