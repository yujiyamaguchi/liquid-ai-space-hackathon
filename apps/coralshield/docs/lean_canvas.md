# CoralShield — Lean Canvas

---

## UVP

> **「NOAAの5km解像度では見えない礁スケールの白化を、Sentinel-2可視光バンドの色変化パターンをLFMが解析することで10m解像度・クリーンシーン取得後に自動検出する（ダイバー目視より早く・NOAA より 100x 高解像度で）」**

## 1. Problem

サンゴ礁の大規模白化・死滅の早期検知ができない。

| 指標 | 数値 |
|---|---|
| サンゴ礁が支える経済価値 | 年間 $375B（観光・漁業・沿岸保護） |
| 過去40年の消滅率 | 50%（Global Coral Reef Monitoring Network） |
| 現在の目視調査 | 1サイト確認に 3〜7日、広域は事実上不可能 |
| NOAA CoralWatch の限界 | 空間解像度 1km（礁スケール 10m の被害を見逃す） |
| 大規模白化頻度 | 以前は10年に1回 → 現在は 2〜3年に1回 |

## 8. Assumptions

1. **[PoC 第一ゲート]** Sentinel-2 B3/B4 比率が GBRMPA 白化座標（2024年）と非白化座標（2023年同月）で統計的に分離できること。分離しない場合は水柱補正（Water Column Correction）の追加実装が必要で /poc2 設計変更が必要
2. 水深 **3〜5m 以浅**の浅礁に限定する（水柱補正の誤差を最小化するため。水深 15m では水柱補正誤差が白化シグナルを上回る可能性がある）
3. 熱帯海域で十分な数のクリーンシーン（雲被覆 < 20%）が取得できる（GBR 周辺の乾季：6〜9月が主ターゲット）
4. 2024年世界規模白化イベントの GBRMPA 公開座標データが FT 用 GT として使える（外部 GT — 自己 GT 問題なし）
5. パラメトリック保険会社が AI 判定をトリガーとして法的に認める（メキシコ MARI 保険が先例）

## 技術スタック

```
SimSat API (Sentinel-2: B2/B3/B4/B8A、浅礁座標)
  → 水面反射補正（Sunglint 除去）
  → 白化候補抽出: B3/B4 比 > 閾値（白化サンゴは青緑化 → B3↑, B4↓）
     ※ 閾値処理でまず候補領域を絞る（計算コスト削減）
  → 疑似カラー合成 [B4, B3, B8A] → 224×224px チップ
  → LFM 2.5-VL-450M (LoRA FT)
     ※ LFM の役割: B3/B4 候補を「砂底 / 太陽グリント / 濁り水 vs 真の白化サンゴ」に分類する
        false positive フィルタ。この文脈判断（"この白い部分は砂地か白化か"）がルールベースでは不可能
  → {"status": "Bleached", "coverage_pct": 42, "confidence": 0.81}
  → GeoJSON（白化マスク）→ GBRMPA / 保険トリガー API へアラート配信
```

GT: 2024年白化ピーク期間（GBRMPA 公開座標）vs 2023年同月（白化なし）の対比（外部 GT）

## /critic 評価

**総合判定: 要修正 — 52点**（2026-04-24 修正済み → **Go — 57点**）

修正内容:
1. ✅ UVP から "リアルタイム" を除去し「クリーンシーン取得後の即時自動判定」に変更
2. ✅ Assumptions に PoC 第一ゲート（B3/B4 分離確認）を追加
3. ✅ LFM 役割を「B3/B4 候補の false positive フィルタ（砂底/グリント/濁り vs 真の白化）」として技術スタックに明記

**強み**
1. 問題の感情的インパクト（$375B 経済価値・50% 消滅・白化頻度 2〜3 年に 1 回）
2. 差別化が明確（NOAA 1km SST → 10m 直接検出、Allen Coral Atlas は静的・年次）
3. GT が外部データで確立（GBRMPA 2024年公開座標、自己 GT 循環問題なし）

**要修正の3点（lean_canvas に反映してから /poc へ）**
1. **PoC 第一ゲート追加**: SimSat GBR 座標で B3/B4 比率の白化/非白化分離を先行確認。分離できなければ水柱補正が必須で /poc2 前に設計変更が必要
2. **LFM 役割を false positive フィルタに特化**: B3/B4 閾値で候補抽出 → LFM が「砂底/太陽グリント/濁り水 vs 真の白化サンゴ」を文脈判断。これが "LFM でなければできない" 唯一の理由
3. **UVP の "リアルタイム" を修正**: 熱帯域の雲カバーを考慮し「クリーンシーン取得後の即時自動判定（ダイバー到着より早い）」に変更
