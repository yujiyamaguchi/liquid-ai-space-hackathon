# HarborLens — Lean Canvas

---

## UVP

> **「衛星エッジ AI が、銀行の担保調査員より先に港湾タンク周辺の荷役活動異常を検知し、融資リスクを自動でフラグアップする」**

---

## 1. Problem

融資銀行・保険会社が「担保物（港湾タンク・コンテナ）の現状」を独立かつタイムリーに確認する手段がない。

| 指標 | 数値 |
|---|---|
| Hin Leong 事件（2020年、シンガポール） | 約 $3.5B の不正担保融資 |
| 青島港事件（2014年） | $10B 超の不正担保 |
| 現状の監査遅延 | 週次〜月次。72時間の遅延が数百万ドルのエクスポージャー差 |

## 7. Channels

- REST API → 銀行・保険の既存 TMS/ERM システム
- Webhook アラート → Slack / メール / SMS
- ダッシュボード（Web）: 港湾拠点の稼働変化トレンド時系列表示

---

## 8. Assumptions

1. Sentinel-2 の 10m 解像度で港湾タンク・荷役設備の稼働変化を識別できる
2. 荷役船・クレーン・作業エリアの稼働変化を可視光 + NIR で検知できる
3. SimSat がシンガポール・ロッテルダム周辺の時系列データを十分な密度で保有している
4. 銀行コンプライアンスが「衛星証跡」を内部審査参考資料として受理する
5. SimSat 時系列の差分を自己 GT として FT 学習データが構築できる

---

## 技術スタック

```
SimSat API (Sentinel-2: B2/B3/B4/B8/B8A/B11/B12、港湾座標)
  → 陸上エリアマスク（NDWI で水域除外）
  → 荷役エリアのタイル切り出し（256×256px）
  → 時系列差分: T0（ベースライン）vs T1（監視日）
  → LFM 2.5-VL-450M (LoRA FT)
  → {"activity_change": "INCREASED", "confidence": 0.87, "alert": true}
  → 変化ありの JSON のみダウンリンク → REST API → 銀行 TMS
```

GT: SimSat 時系列差分（自己 GT）または公開港湾統計との定性照合
