## UVP（確定）

> **「Sentinel-2 の植生乾燥指標 (NDMI_p10) を LFM が時系列解析し、21 日前から乾燥傾向を追跡・7 日前に高リスクを確定。既存データの 4〜5 倍の解像度（20m）で南向き斜面単位のリスクを特定する。保険会社は正確な引受判断を、消防・林業機関は事前対策の根拠を得る」**

**スコープ制約:**
- FireGuard は**発火前の燃料リスク評価**に特化
- 評価対象: **植生含水率・乾燥度（燃料条件）**。気象リスク（湿度・風・温度）は Sentinel-2 から得られないため対象外。「FireGuard のスコア = 燃料条件スコア」として位置づける

**FireEdge からの継承:**
- FireEdge（同一リポジトリ内）が Sentinel-2 + LFM FT の有効性を PoC2 で定量的に実証済み
- GT パイプライン（FIRMS 発火座標 × SimSat 画像取得）・FT インフラ（dataset_builder / train / evaluate）を FireGuard で直接再利用する
- 「LFM FT が Sentinel-2 画像で機能する」という本アプリの技術前提は FireEdge PoC2 によって既に裏付けられている

## 1. Problem（課題）

| 指標 | 数値 |
|---|---|
| 2023年カナダ山火事 | 1,840万ha 焼失・$20B 超の損害 |
| カリフォルニア山火事保険危機 | 2023年、主要保険7社が引受停止。State Farm だけで 7.2 万件解約 |
| 既存リスクデータの解像度 | CoreLogic / Verisk 等は 90〜250m。丘の南斜面と北斜面を区別できない |
| 予防的野焼きの効果 | 計画的野焼き実施エリアの延焼リスク 60〜90% 低下（CSIRO 研究） |

- **保険会社の課題**: 既存リスクデータ（CoreLogic / Verisk 等）が粗すぎて「引き受けるべき土地・引き受けない土地」を精密に判断できない。その結果、市場全体から撤退せざるを得なくなっている
- **行政・消防の課題**: 計画的野焼きの実施場所・優先順位付けに使えるリスクマップが 1km スケールしかなく、具体的な丘・斜面の単位で判断できない
- **既存衛星リスクシステムの限界**: Copernicus EFFIS は欧州限定・1km。MODIS NDVI は 250〜500m。「どのエリアが具体的に危ないか」の特定が不可能

**Full Customer Segment:**
- P&C 損害保険会社（山火事リスクの引受・プレミアム算定）
- 消防局・林野庁（計画的野焼き・資機材事前配備の計画立案）
- 民間林業会社・太陽光/風力発電会社（自社資産エリアのリスク把握）
- 再保険会社（Munich Re / Swiss Re 等：再保険プライシング）

**Early Adopter（現実的な初期顧客）:**
1. **P&C 損害保険会社**（Swiss Re、Tokio Marine、State Farm 等）
   - 山火事保険引受の精度向上に 20m リスクマップが直接使える（CoreLogic 90〜250m の 4〜12 倍精細）
   - CoreLogic / Verisk 等の既存リスクデータに追加費用として正当化しやすい
   - 民間企業のため調達サイクルが短い（四半期以内の PoC 採用も可能）
   - カリフォルニア危機で「より良いリスクデータ」への需要が最高潮
2. **再保険会社**（Munich Re / Swiss Re）
   - 世界規模のカタストロフリスク評価に使える
   - 自社 R&D 予算でデータ購入・試験導入が速い
3. *消防局・林野庁は予算はあるが政府調達サイクル（2〜5年）のため後期フェーズ*

## 4. Unfair Advantage（圧倒的優位性）

| 競合 | 解像度 | 更新頻度 | 地域 | エッジ処理 | FireGuard 優位性 |
|---|---|---|---|---|---|
| **FireGuard** | **20m** | **5日** | **地中海性気候帯** | **あり** | — |
| CoreLogic / Verisk | 90〜250m | 年次〜月次 | 米国中心 | なし | **4〜12x 高解像度・更新頻度** |
| Copernicus EFFIS | 1km | 週次 | 欧州のみ | なし | **50x 高解像度・全球対応** |
| FWI（気象指数） | 10〜25km | 日次 | 全球 | なし | **空間的特定（丘・斜面単位）が可能** |
| MODIS/VIIRS ベース | 250〜500m | 日次 | 全球 | なし | **12〜25x 高解像度 + LFM 時系列推論** |
| Planet Analytics / Descartes Labs | 3〜10m | 日次〜週次 | 全球 | **なし（地上処理）** | **オンボード処理で 1〜3 日早く配信。LFM 時系列パターン認識（RF 非対応）** |

**FireGuard のみが持つ組み合わせ:**
- **20m 解像度（B8A/B11）**: CoreLogic 90〜250m の 4〜12 倍。丘の南斜面と北斜面を区別できる最初の商用リスクデータ。EFFIS 1km の 50 倍。※ NDMI 計算に使う B8A・B11 は 20m ネイティブ解像度
- **LFM の固有タスク: 時系列パターン認識**: 単一シーンの分光分類（Random Forest で代替可能）ではなく、「3〜5 期の時系列で植生がどのように変化してきたか」というシーケンシャルパターンを画像として認識し、過去の火災前シグナルと照合する。このタスクはピクセル単位の RF には困難で、VLM の視覚的時系列推論が本質的に必要
- **オンボード処理によるリアルタイム配信**: Planet Analytics / Descartes Labs は同じ 10m 衛星データを地上処理するため配信に 1〜3 日遅れる。FireGuard はダウンリンク即時に高リスク GeoJSON を配信可能
- **帯域削減 90%+**: 低リスク正常シーンを廃棄し高リスクタイルのみ送信

## 5. Key Metrics（成功指標）

**精度目標（顧客 = 損害保険会社の引受判断を想定）:**

| 指標 | 目標値 | 根拠 |
|---|---|---|
| 高リスクエリア Recall | ≥ **0.80** | 実際に燃えた場所を 2 週間前に特定できた割合 |
| False Positive Rate | ≤ **0.20** | 誤フラグが多いと保険会社が使えない |

| Lead time (傾向トリアージ) | **14〜21 日前** | NDMI_p10 p=0.081 △ / NDMI_mean p=0.093 △ — 早期警戒フラグ。保険引受・計画的野焼きの意思決定はこのフェーズで |

- 推論時間: < 30秒/シーン
- 帯域幅削減率: ≥ 90%（低リスクシーンを廃棄）

## 8. Assumptions（前提仮説・未検証）

1. **[PoC 第一ゲート]** ✅ **完了 (2026-05-02)** 発火 7〜21 日前に Sentinel-2 の NDMI_p10 が統計的に有意なシグナルを示すこと → **NDMI_p10 -7d: Δ=-0.083, p=0.038 (p<0.05)** / -14d: p=0.081 (△) で達成。対象: カリフォルニア チャパラル n=10。
2. **燃料リスク限定**: 気象（湿度・風・温度）は Sentinel-2 から得られない。FireGuard は「燃料条件スコア」に特化し、気象データとの統合は将来拡張として明記する
3. **GT 前提**: FIRMS「燃えた座標・燃えた日時」を遡り、発火 **7 / 14 / 21** 日前の SimSat 画像をラベル付けできること（FireEdge で実装済みの FIRMS × SimSat パイプラインを再利用）。-28d はシグナルなし (p=0.357) のため除外。
4. **FT 有効性前提**: LFM 2.5-VL-450M の LoRA FT が Sentinel-2 画像で有効であること → **FireEdge PoC2 で定量的に実証済み**（ベースモデル比で Recall・F1 改善を確認）
5. **市場前提**: 損害保険会社が「既存 CoreLogic データ（90m）を 20m に高精度化する」ことに年間数百万円〜の追加コストを承認する
6. **偽陽性フィルタ**: LFM が「農地・日陰・刈り取り後」の低 NBR を高リスクと誤判定しないこと（FT で学習可能）

## エッジ処理の必然性

- 山火事シーズン（北半球は 6〜9月、南半球は 11〜3月）は監視エリアが広大
- 全量地上送信 → 処理遅延 + 帯域幅コスト高
- オンボード処理: NBR/NDVI を軌道上で即算出 → 低リスクシーン（正常植生）を廃棄 → **高リスクエリアの GeoJSON のみ送信（推定 10〜50KB/イベント）**
- これにより「5日ごとの全球リスク更新」を現実的な帯域幅で実現

## デモシナリオ（固定）

**ストーリー**: 「2018年11月 カリフォルニア Woolsey Fire（マリブ〜サウザンドオークス）。焼失 97,000 エーカー・3 名死亡・推定損害 $6B。発火地点を SimSat で遡ると、発火の **14〜7 日前** にかけて NDMI_p10 が -0.257（非発火チャパラル平均 -0.087 を大きく下回る）まで低下しており、FireGuard が高リスクフラグを送信していた。保険会社はマリブ沿岸エリアの新規引受を事前に停止できた」

> **なぜ Woolsey か**: PoC で NDMI_p10 -7d = -0.257（NEG mean との差 Δ = -0.170）と最も明確なシグナルを示したイベント。マリブ高額住宅地が対象 = 保険会社のビジネスインパクトが最大。CZU は沿岸霧の影響で NDMI が逆方向に動く例外ケースであり、デモには不適。

> **なぜカリフォルニア・チャパラルか**: 地中海性気候帯チャパラルは PoC で NDMI_p10 p=0.038 を達成した唯一の植生タイプ。針葉樹は NDMI 変動幅が小さく分離不可（p=0.467）。デモシナリオはチャパラル限定で固定する。

| ステップ | Before（FireGuard なし） | After（FireGuard あり） |
|---|---|---|
| 10/25（発火 14 日前） | リスク不明（次の CoreLogic 更新は数ヶ月後） | NDMI_p10 低下傾向 → 早期警戒フラグ（保険会社への通知） |
| 11/1（発火 7 日前） | リスク不明のまま | NDMI_p10 = -0.257 → 高リスク確定フラグ送信 |
| 保険会社の判断 | 当該エリアの保険継続 | 新規引受を停止・既存契約者に避難勧告準備 |
| 11/8（Woolsey Fire 発生） | 多数の保険金請求 → 損失 | 引受済み契約を最小化済み |

## 技術スタック

```
SimSat API (Sentinel-2: B8A/B11/B12、チャパラル山火事多発エリア座標)
  → NDMI_p10 = 10th-pct[(B8A - B11)/(B8A + B11)]  ← 主指標（PoC実証済み）
      B8A: 865nm 近赤外, B11: 1610nm 液体水吸収帯
  → 疑似カラー合成: R=B12, G=B8A, B=B11 → 250×250px PNG (196 VLM tokens)
  → 時系列3点: -21d → -14d → -7d の変化パターン
  → LFM 2.5-VL-450M (LoRA FT)
     ※ LFM の役割: 農地・日陰・岩盤露出等の低 NDMI を除外し
       「先行乾燥スポットが時系列で悪化中」という本当の高リスク可燃物を特定
  → {"risk_level": "HIGH", "ndmi_p10": -0.17, "confidence": 0.84}
  → 高リスクエリアの GeoJSON のみダウンリンク → 保険会社 API / 消防ダッシュボード
```

**GT 構築フロー:**
```
FIRMS アーカイブ（発火座標・日時）
  → 発火 -7日 / -14日 / -21日 前の SimSat シーンを取得（-28d は除外）
  → POS ラベル: 「この画像は X日後に燃えた」
  → NEG ラベル: 同季節・同気候帯の非発火チャパラル地点の画像
  → 感度分析: 何日前まで有意なシグナルが取れるか（PoC 第一ゲート）
```
