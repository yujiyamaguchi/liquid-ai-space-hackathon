# FireEdge — 火災検知スペクトル科学

FireEdge 固有のスペクトル知識です。共通基礎は `docs/spectral_background.md` を参照。

---

## 1. なぜ SWIR2.2 で火災が「光る」のか

**プランク放射の原理:**
物体は温度に応じた電磁波を放射します（黒体放射）。
- 常温の地表（300K）: 放射のピークは遠赤外域（10μm付近）
- 活発な火災（600〜1200K）: ピークは遠赤外だが、**2.2μm でも非常に強い放射が発生**
- 通常の植生・土壌（300K）: 2.2μm での熱放射はほぼゼロ

→ SWIR2.2 (B12) の値が突出して高いピクセル = 高温物体（≒火災）

---

## 2. なぜ煙を「透過」できるのか

**ミー散乱の原理:**
- 煙粒子の典型サイズ: **0.1〜1μm**
- 可視光 (0.5μm): 粒子サイズと同程度 → **強く散乱 → 煙が不透明**
- SWIR2.2 (2.2μm): 粒子より長い → **散乱が弱い → 透過**

> **注意**: VIIRS M13 (3.75μm) の方が SWIR2.2 より煙透過性は高い。
> FireEdge の優位性は「煙透過」ではなく「20m 高解像度 × オンボードAI推論」。

---

## 3. スペクトル指標

### NBR2（活火災検出に最適）
```
NBR2 = (SWIR1.6 - SWIR2.2) / (SWIR1.6 + SWIR2.2)
```
| 状態 | NBR2 | 理由 |
|---|---|---|
| **活火災** | << 0（例: -0.3〜-0.8） | SWIR2.2 が急上昇 |
| 焼失跡 | 軽度の負（-0.05〜-0.2） | |
| 健全植生 | 正（+0.2〜+0.5） | 水分でSWIR1.6 > SWIR2.2 |
| 裸地 | ≒ 0 | |

**判定式**: `NBR2 < -0.05 AND SWIR2.2 > 0.15` → 活火災

### NBR（焼失跡検出）
```
NBR = (NIR - SWIR2.2) / (NIR + SWIR2.2)
```

## 4. SWIR 疑似カラー合成

```
R = SWIR2.2 (B12) → 活火災が赤〜橙
G = SWIR1.6 (B11) → 熱異常が黄緑
B = NIR (B8A)      → 健全植生が青
```

| 地物 | 疑似カラーでの見え方 |
|---|---|
| **活火災** | **明るい赤〜橙** |
| 健全植生 | 青〜青緑 |
| 裸地 | 黄〜茶 |
| 水域 | 暗い |
| 煙 | 灰色〜暗い |

## 5. 雲 vs 煙（重要な制約）

| | 粒子サイズ | SWIR2.2 透過性 |
|---|---|---|
| 煙 | 0.1〜1μm | **透過できる** |
| 雲（水滴） | 5〜100μm | **透過できない** |

**FireEdge の前提**: 観測シーンに雲がないこと。乾季・高気圧下の火災シナリオが主ターゲット。

---

### 5a. 雲量フィルター（データ収集ポリシー）

`finetune/dataset_builder.py` では `max_cloud_cover = 50%` を閾値としてシーンを除外している。

**閾値の根拠:**
- SimSat が返す `cloud_cover` は Sentinel-2 の scene-level 雲量推定値
- 50% 超の場合、火災域を含む画像領域が雲で遮蔽されている可能性が高い
- 雲下の burn scar は SWIR でも検出不可能（雲は不透過）→ ラベルが不正確になる

**データ収集への影響（/finetune 実測値）:**

| カテゴリ | 試行 | 取得成功 | 雲量超過スキップ | 取得率 |
|---|---|---|---|---|
| 砂漠・乾燥地 | 10 | 5 | 5 | 50% |
| 都市郊外 | 8 | 4 | 4 | 50% |
| 湿地・デルタ | 8 | 5 | 3 | 63% |
| サバンナ | 8 | 3 | 5 | 38% |
| 温帯森林 | 6 | 4 | 2 | 67% |
| 農地・草地 | 5 | 4 | 1 | 80% |
| **合計** | **45** | **25** | **20** | **56%** |

雨季・季節に当たるカテゴリ（サバンナ rainy season、熱帯）のスキップ率が高い。
これはタイムスタンプを一時点に固定したためであり、実運用では軌道周期内で最も雲量の低い時刻を選択できる。

**モデルバイアスの含意:**

> **雲量フィルターは「晴天偏向（clear-sky bias）」を生む。**

- 訓練・評価データはすべて雲量 ≤ 50% のシーンで構成される
- 実運用でも光学センサーが機能するのは晴天時のみ（雲に覆われたシーンでは「検出不可」を返すべき）
- したがって clear-sky bias は **仕様上の前提と一致しており、バグではない**
- ただし雲量 30〜50% のシーンでは部分的な雲遮蔽がある可能性があり、そのようなシーンでの精度は低下しうる

**運用上の対処:**
- 推論パイプラインは `cloud_cover > 50%` の場合に推論をスキップし、`"CLOUDY — inference skipped"` を返す（実装済み: `src/data_fetcher.py`）
- これにより誤検知リスクを排除し、システムとして「雲量が低い晴天時のみ高精度で機能する」ことを明示する

## 6. FIRMS/VIIRS との比較

| | Sentinel-2 B12 (SWIR2.2) | VIIRS M13 (FIRMS) |
|---|---|---|
| 中心波長 | 2.19μm | 3.75μm |
| 空間解像度 | **20m** | 375m |
| 煙透過性 | 高い | より高い（長波長） |
| 観測時間帯 | **昼間のみ**（地方時 ~10:30 通過） | 昼夜複数回（熱赤外で夜間検知可能） |
| 用途 | **高解像度・通過時即時AI判断** | 広域・夜間含む継続監視 |
| 再訪問周期 | 2〜5日 | 6〜12時間（3衛星） |

**昼間専用観測（重要）**: Sentinel-2 MSI は受動光学センサーであり、太陽光の反射を検出する。太陽同期軌道により地方時 ~10:30 に通過し、夜間は撮像データを収集しない。夜間発生した火災は次回昼間通過（1〜5日後）まで検出できない。

**役割の補完関係**: FireEdge は VIIRS を置き換えるのではなく補完する。VIIRS が「昼夜の広域センシング」を担い、FireEdge が「通過時の20m高解像度 × VLM文脈推論」を担うアーキテクチャが現実的。

**遅延の注意**: FIRMS NRT は 9〜15時間（再訪問+処理）。Sentinel-2 は軌道周期が 2〜5日のため、トータル遅延では不利。FireEdge の優位性は遅延でなく **解像度・オンボード処理・VLM推論**。
