# FireEdge — Lean Canvas
**/ideate 完了: 2026-04-18**

## UVP（確定）

> **「地上処理なしに、衛星が通過した瞬間に20m精度のSWIRとAI推論で火災を自律判断し、2KBでアラートを通知する」**

---

## 1. Problem（課題）

- 野火は毎年拡大。2023年カナダだけで1,840万ha焼失（史上最大）
- 初期検知の遅れが延焼の主因。既存手段の限界:
  - 地上目視: 煙で視界ゼロ、人員が届かない広大な山林
  - FIRMS/VIIRS: 375m解像度で初期小火種を見逃す。処理後 9〜15時間の遅延
  - GOES/Himawari: 解像度 ~2km。広域監視には使えるが精密位置特定に不向き

---

**Full Customer Segment:**

## 4. Unfair Advantage（圧倒的優位性）

| 競合 | 弱点 | FireEdge の優位性 |
|---|---|---|
| FIRMS/VIIRS | 375m低解像度、閾値処理。昼夜複数回通過・夜間熱赤外検知可能 | **20m解像度 + VLM文脈推論**（昼間通過時限定。夜間はVIIRSが補完） |
| OroraTech（最大直接競合） | AI使用だが地上処理前提。地上局必要 | **オンボード完結（地上局不要）** |
| GOES/Himawari | 静止軌道で速いが ~2km解像度 | **20m解像度** |
| Planet Labs | RGB のみ（熱感知不可） | **SWIR熱異常検知** |

**FireEdge のみが持つ組み合わせ:**
- **オンボード処理**: 地上局なしでアラートを生成・保持。接触時に即送信
- **VLM文脈推論**: 「計画的野焼き vs 山火事」の区別、延焼方向・重大度の言語出力（閾値処理には不可能）
- **20m高解像度**: FIRMS 375mピクセルで平均化されて消える初期小規模火種を検出
- **エッジ最適化LFM**: 450M params / < 1GB VRAM — 衛星搭載計算予算に収まる唯一級のVLM

**重要な注意**: 「煙透過」は主要差別化軸として使わない。VIIRS (3.75μm) の方が煙透過性は高い。

> **VIIRS との補完関係（昼間専用観測の制約）**: Sentinel-2 は地方時 ~10:30 の昼間通過のみ観測可能（受動光学センサーのため夜間不可）。VIIRS は昼夜複数回通過し熱赤外で夜間も検知できる。FireEdge は VIIRS の「広域・夜間センシング」を置き換えるのではなく、「通過時の高解像度・即時AI判断」で補完するポジション。

## 5. Key Metrics（成功指標）

**精度目標（顧客=林野庁・消防を想定、Recall優先）:**
- Recall > **0.85**
- Precision > 0.70
- False Positive Rate < **0.15**
- F1 > 0.75

> **スコープ注記（センサー物理限界）**: FRP 下限は設けない。ただし低 FRP（< 20MW 程度）の
> 小規模火災は Sentinel-2 SWIR (20m) の混合ピクセル問題により光学シグナルが希釈され、
> 検知困難な場合がある（混合ピクセル問題）。一方、FRP が低くても燃焼面積が十分なケースでは
> Sentinel-2 でも活火災シグナルが確認できるため、FRP 閾値では一律にカットしない。
> 小規模焼畑など検知できないケースは VIIRS 補完に委ねる。

**技術指標:**
- 推論時間: < 10秒（衛星通過中に完了）
- アラートサイズ: < 2KB
- VRAM: < 1GB（衛星エッジ予算内）
- 空間解像度: 20m (Sentinel-2 B12)

## 8. Assumptions（前提仮説・未検証）

1. **技術前提**: LFM Fine-Tuningにより SWIR疑似カラーでの火災検知精度が閾値処理を超える（→ /poc2で検証）
2. **データ前提**: FIRMS座標でSimSatが有効なSentinel-2データを返す。スペクトルシグナルが fire/no-fire で分離できる（→ /poc で検証）
3. **事業前提**: 衛星オペレーターがオンボードAI搭載に意欲的（ハッカソン与件）
4. **制約前提**: **観測シーンに雲がないこと**（煙は透過できるが雲は光学センサーで不透過。乾季・高気圧下の火災シナリオが主ターゲット）。また Sentinel-2 は地方時 ~10:30 の**昼間通過のみ観測可能**（受動光学センサーのため夜間は撮像しない）。夜間発生の火災は次回通過（1〜5日後）まで検出できない
5. **市場前提**: 早期採用者が価値を認め、数百万円〜数千万円規模の初期契約が取れる
6. **技術依存**: FireEdge は衛星を持たない。衛星オペレーターが搭載・展開パートナーとなる

## /critic 評価・位置づけ変更（2026-04-25）

**当初判定: Go — 69点 → 再評価: 技術デモ — 58点**

### UVP が成立しない理由（最終判断）

1. **5〜10 日リビジット vs VIIRS 1日 2〜4 回**: 火災は既に VIIRS が検知済みの状態で Sentinel-2 が通過する。「早期検知」は成立しない
2. **野焼き vs 山火事の判別が未検証**: PoC2 で検証したのは「FIRMS 確認済み火災 vs 非火災」の2クラスのみ。野焼きをネガティブサンプルとした学習は未実施
3. **Burn scar mapping に転換しようとしたが GT が成立しない**: MODIS 500m（解像度不足）・dNBR 閾値（循環参照）・手動注釈（工数超過）のいずれも実用的でない

### FireEdge が FireGuard に継承するもの

FireEdge の技術成果は破棄しない。**FireGuard の FT 根拠として活用する。**

| 資産 | 継承先での用途 |
|---|---|
| Sentinel-2 SWIR パイプライン（SimSat API → 疑似カラー合成） | FireGuard の NIR/SWIR パイプラインに転用 |
| LFM 2.5-VL-450M LoRA FT インフラ（dataset_builder・train・evaluate） | FireGuard の FT にそのまま使用 |
| FIRMS GT パイプライン（発火座標 × SimSat 画像取得） | FireGuard の「発火 N 日前シーン取得」に直接再利用 |
| **PoC2 での定量的実証**（FT 有効性確認・LoRA で few-shot 超え） | 「LFM FT が Sentinel-2 画像で有効」という FireGuard の技術前提を裏付ける |
