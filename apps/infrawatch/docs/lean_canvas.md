# InfraWatch — Lean Canvas

---

## UVP

> **「衛星が洪水後 5〜10 日以内に道路ネットワークの浸水・損傷リスクを自律スコアリングし、損害保険会社の査定を人手なしで自動化する」**

---

## 1. Problem

大規模災害後、橋梁・主要道路の通行可否確認に 24〜72 時間かかる。その間、緊急車両・支援物資の動線が不明のまま。

| 指標 | 数値 |
|---|---|
| 1橋崩落の経済損失 | 平均 $500M〜$2B（迂回・物流停止含む） |
| 年間洪水被害（世界） | $82B（Swiss Re, 2023） |
| 問題の本質 | 道路が壊れているから点検できない、の逆説 |

---

## 2. Customer / Early Adopter

| 層 | 組織 | 購買動機 |
|---|---|---|
| **Early Adopter** | **自然災害保険の査定部門（Munich Re、Tokio Marine）** | AI 判定を参考情報として使える。査定速度改善で即課金可 |

| Customer Segment 3 | 国連 OCHA・FEMA 等の人道支援機関 | 孤立集落への支援ルート選定 |

## 4. Unfair Advantage

| ソリューション | 検知時間 | 空間解像度 | 自動化度 |
|---|---|---|---|
| **InfraWatch** | **発災後 5〜10日（クリーン画像取得後）** | 10m | **完全自動** |
| Copernicus EMS | 大規模イベント年間数十件のみ対応。手動判読・数日遅延 | 10m | 半手動 |
| ドローン点検 | 24〜72h（天候依存、浸水地域へのアクセス困難） | 2〜5cm | 半自動 |
| 航空写真解析 | 12〜48h | 30cm | 半手動 |
| 従来衛星画像（地上処理） | 12〜36h（雲晴れ後） | 10m | 部分自動 |

**InfraWatch のみが持つ組み合わせ:**
- **自動化 × 網羅性**: Copernicus EMS がカバーしない中小規模洪水（年間数千件）を自動処理。EMS は人道支援規模のイベントのみ起動
- **保険査定ユースケース特化**: 発災後 5〜10 日は道路の緊急対応が落ち着き、保険会社が現地調査を開始するフェーズ。このタイミングに InfraWatch の画像が届くことが価値
- **LFM severity 判断**: NDWI が "浸水あり" を出した後、LFM が「軽微な表面浸水（通行可能）vs 構造的損傷の可能性（通行不能）」を文脈判断。この重大度分類はルールベースでは不可能

## 8. Assumptions

1. Sentinel-2 の 10m 解像度で道路・橋梁の水没（浸水判定）を識別できる
2. NDWI + 道路レイヤー重ね合わせで通行不能ゾーンが抽出できる
3. Copernicus EMS アクティベーション座標が FT 用 GT として使える
4. 損害保険会社が「AI 参考情報」として査定資料に活用できる規制環境にある
5. SimSat が洪水被災座標・日時のデータを保有している

## 技術スタック

```
SimSat API (Sentinel-2: B4/B3/B8/B11/B12)
  → NDWI = (B3-B8)/(B3+B8) → 水域マスク（浸水候補エリア抽出）
  → 道路ネットワーク（OpenStreetMap）との重ね合わせ → 浸水道路セグメント候補
  → 疑似カラー合成 [B12, B11, B4] → 256×256px タイル
  → LFM 2.5-VL-450M (LoRA FT)
     ※ LFM の役割: NDWI が検出した浸水を「軽微な表面浸水（乾いてきている）vs 継続的浸水・構造的損傷」に severity 分類
        → この文脈判断がルールベースでは不可能（水の残留パターン・土砂堆積・構造変化を画像から読む）
  → {"risk_score": 0.87, "status": "impassable", "severity": "structural_damage", "confidence": 0.91}
  → GeoJSON（通行不能リスクマップ）→ 保険会社査定システムへダウンリンク
```

GT: Copernicus EMS アクティベーション座標（大規模イベント用）+ 中小規模洪水は現地新聞報道 / OSM 道路閉鎖データで補完
