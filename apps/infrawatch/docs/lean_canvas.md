# InfraWatch — Lean Canvas

---

## UVP

> **「衛星が軌道上で洪水後の道路ネットワークの通行不能リスクを自律スコアリングし、軌道通過ごとに回避ルートマップを地上に届ける」**

---

## 1. Problem

大規模災害後、橋梁・主要道路の通行可否確認に 24〜72 時間かかる。その間、緊急車両・支援物資の動線が不明のまま。

| 指標 | 数値 |
|---|---|
| 1橋崩落の経済損失 | 平均 $500M〜$2B（迂回・物流停止含む） |
| 年間洪水被害（世界） | $82B（Swiss Re, 2023） |
| 問題の本質 | 道路が壊れているから点検できない、の逆説 |

---

## 8. Assumptions

1. Sentinel-2 の 10m 解像度で道路・橋梁の水没（浸水判定）を識別できる
2. NDWI + 道路レイヤー重ね合わせで通行不能ゾーンが抽出できる
3. Copernicus EMS アクティベーション座標が FT 用 GT として使える
4. 損害保険会社が「AI 参考情報」として査定資料に活用できる規制環境にある
5. SimSat が洪水被災座標・日時のデータを保有している

## 技術スタック

```
SimSat API (Sentinel-2: B4/B3/B8/B11/B12)
  → NDWI = (B3-B8)/(B3+B8) → 水域マスク
  → 道路ネットワークとの重ね合わせ
  → 疑似カラー合成 [B12, B11, B4]
  → LFM 2.5-VL-450M (LoRA FT)
  → {"risk_score": 0.87, "status": "impassable", "confidence": 0.91}
  → GeoJSON（通行不能リスクマップ）→ ダウンリンク
```

GT: Copernicus EMS アクティベーション座標
