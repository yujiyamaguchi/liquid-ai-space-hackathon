# Consistency Check

プロジェクト全体のドキュメント・コード間の整合性をチェックし、不整合をレポートする。
修正は**ユーザーの承認を得てから**行う。自動修正しない。

## スキャン対象

含める:
- `CLAUDE.md`, `README.md`
- `apps/comparison.md`
- `apps/*/docs/*.md`
- `apps/*/src/*.py`, `apps/*/finetune/*.py`
- `.claude/commands/*.md`

除外: `.venv/`, `__pycache__/`, `data/`, `output/`, `simsat/`

## 手順

### Step 1: 方針文書を読む
`CLAUDE.md` と `README.md` を読み、現在の方針・概念・定数を把握する。
特に「削除・変更された原則」「定義されている用語」を抽出する。

### Step 2: 対象ファイルを全件読む
スキャン対象の全ファイルを読み込む。

### Step 3: 2種類のチェックを行う

**A. 数値・固有値の不一致**
同じ意味を持つ値（Criticスコア・フェーズ名・モデル名・ポート番号・
APIエンドポイント・定数値等）が複数ファイルに出現している箇所を探し、
値が食い違っていないか確認する。

**B. ゾンビ参照**
CLAUDE.md・README.md で削除・変更された概念・用語（例: "SDD"）を
プロジェクト全体で検索し、現在の方針と矛盾する残存箇所を検出する。

### Step 4: レポートを出力する

以下の形式で出力する:

```
## Consistency Check Results

### A. 数値・固有値の不一致
1. [HIGH/MED/LOW] <項目名>
   - <ファイル A:行番号>: <値>
   - <ファイル B:行番号>: <値> ❌ / ✅

### B. ゾンビ参照
2. [HIGH/MED/LOW] "<用語>" への参照
   - 現在の方針: <CLAUDE.md 等での現状>
   - <ファイル:行番号>: <該当箇所> ❌

---
修正してよい項目の番号を教えてください。
整合性として問題ない場合は「問題なし」と教えてください。
```

### Step 5: 承認された項目のみ修正する
ユーザーが承認した番号の項目だけを修正する。
修正内容を変更前に確認として示してから実行する。
